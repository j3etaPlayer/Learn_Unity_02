# 24.03.11 게임 엔진에 대하여

## 게임은 어떻게 만들어야 하는가

유니티는 컴포먼트 기반 스크립팅을 제공한다.

### A. 컴포넌트(Component)란?
 - 정의 : 소프트웨어 구성 단위 - 모듈(module)
 - 이론 : 필요한 프로그램을 개발 할 때 이 모듈들을 조립해서 개발 할 수 있다.

### B. 컴포넌트 기반 프로그래밍 방식
 - 객체를 카테고리화해서 컴포넌트화 시켜 재사용한다.
 - 컴포넌트를 조립하는 뼈대에 컴포넌트를 조립하여 객체를 구현한다.
 - 뼈대(Entity), 유니티 - GameObject

### C. 유니티에서 컴포넌트화된 요소
 - Position Component       : 게임 오브젝트들의 위치, 방향, 크기를 결정하는 컴포넌트
 - Rendering Component      : 가상 환경에 있는 오브젝트를 실제로 화면에 보이도록 처리되는  
                              프로세스를 렌더링이라 하며, 이 렌더링을 할 때 필요한 컴포넌트
 - Physics Component        : 게임에서 물리기능을 담당하는 컴포넌트. RigidBody(3d, 2d)
 - Input Component          : 유저의 입력을 받아 처리하는 컴포넌트
 - Network Component        : 네트워크 처리를 담당하는 컴포넌트

### D. 유니티 스크리트 방식
 - Unity에서 스크립트를 생성하면 Monobehavior를 상속 받는 상태로 불려진다.
 - 클래스 생성 시 유니티의 컴포넌트 방식을 사용하여 오브젝트에 컴포넌트를 부착할 수 있고,
   Monobehavior가 제공하는 이벤트 함수를 사용할 수 있게 된다.

# 24.03.15 물리와 충돌
 - Rigidbody
  1. 게임오브젝트에 Rigidbody컴포넌트를 추가하여 물리적 동작이 가능하게 한다.
  2. 충돌체(Collider) 컴포넌트가 하나 이상 존재할 경우 게임 오브젝트에 발생하는 충돌에 의해 움직인다.
 - Collider : 물리 충돌 처리를 위해 존재하는 컴포넌트이다. 게임에서 눈에 보이지 않으며 정확하게 오브젝트와 같은 모양일 필요가 없으며, 근사치로 표현하여 게임의 성능을 올리는 방식으로 구현되고 있습니다.

 - 물리 충돌 이벤트(스크립트로 작동되는 엑션)

  1. collition 이벤트  : 두 오브젝트 사이에 물리적인 현상이 발생하고 오브젝트가 갖고 있는 컴포넌트에 따라서 물리 연산 검사가 발생한다.
  2. trigger 이벤트 : trigger로 설정된 오브젝트를 통과하고 트리거의 온/오프를 통해서 스크립트를 작동시키는 검사가 발생한다.

 - OnCollision Enter, Stay, Exit 이벤트 함수
  1. Event  : 물리 충돌이 발생했을 때 이벤트 발생
  2. Stay   : 물리 충돌 중일때 계속해서 이벤트 발생
  3. Exit   : 물리 충돌에서 벗어났을 때 이벤트 발생
