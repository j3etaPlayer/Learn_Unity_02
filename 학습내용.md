# 24.03.11

## 게임은 어떻게 만들어야 하는가

유니티는 컴포먼트 기반 스크립팅을 제공한다.

### A. 컴포넌트(Component)란?
 - 정의 : 소프트웨어 구성 단위 - 모듈(module)
 - 이론 : 필요한 프로그램을 개발 할 때 이 모듈들을 조립해서 개발 할 수 있다.

### B. 컴포넌트 기반 프로그래밍 방식
 - 객체를 카테고리화해서 컴포넌트화 시켜 재사용한다.
 - 컴포넌트를 조립하는 뼈대에 컴포넌트를 조립하여 객체를 구현한다.
 - 뼈대(Entity), 유니티 - GameObject

### C. 유니티에서 컴포넌트화된 요소
 - Position Component       : 게임 오브젝트들의 위치, 방향, 크기를 결정하는 컴포넌트
 - Rendering Component      : 가상 환경에 있는 오브젝트를 실제로 화면에 보이도록 처리되는 프로세스를 렌더링이라 하며,
                              이 렌더링을 할 때 필요한 컴포넌트
 - Physics Component        : 게임에서 물리기능을 담당하는 컴포넌트. RigidBody(3d, 2d)
 - Input Component          : 유저의 입력을 받아 처리하는 컴포넌트
 - Network Component        : 네트워크 처리를 담당하는 컴포넌트

### D. 유니티 스크리트 방식
 - Unity에서 스크립트를 생성하면 Monobehavior를 상속 받는 상태로 불려진다.
 - 클래스 생성 시 유니티의 컴포넌트 방식을 사용하여 오브젝트에 컴포넌트를 부착할 수 있고,
   Monobehavior가 제공하는 이벤트 함수를 사용할 수 있게 된다.
